Detailed Description

The menu.

Initializes and displays the menu.
Provides the function MENU_check_transition() for polling user actions. The variable MENU_transition is set to the touched menu item. If no touch has occurred the variable MENU_transition is set to MENU_NONE
If the interrupt handler is enabled by calling BSP_TS_ITConfig(); the variable MENU_transition is set to the touched menu entry as above.
Either call once BSP_TS_ITConfig() to enable the interrupt or MENU_check_transition() in the main while loop for polling.
The function MENU_get_transition() returns the new menu item.
MENU_values_act(int16_t x_distance, uint16_t y_distance, int16_t angle, float current) and MENU_visual_act(int16_t x_distance, uint16_t y_distance, float current) display show the orientation to the cable.

void EXTI15_10_IRQHandler ( void  )

Interrupt handler for the touchscreen.

Note
BSP_TS_ITConfig(); must be called in the main function to enable touchscreen interrupt.
There are timing issues when interrupt is enabled. It seems that polling is the better choice with this evaluation board.
Call MENU_check_transition() from the while loop in main for polling.

The touchscreen interrupt is connected to PA15.
The interrupt handler for external line 15 to 10 is called.

void MENU_check_transition ( void  )

Check for selection/transition.

If the last transition has been consumed (MENU_NONE == MENU_transition) and the touchscreen has been touched for a defined period the variable MENU_transition is set to the touched item.

Note
Evalboard revision E (blue PCB) has an inverted y-axis in the touch controller compared to the display. Uncomment or comment the #define EVAL_REV_E in main.h accordingly.
